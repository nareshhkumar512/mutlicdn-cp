The problem

Akamai, Cloudflare, and CloudFront express the same delivery intent in very different ways. Load balancing, rules, cache behavior, certificates, and security controls do not line up cleanly provider to provider.

Today, provider-native automation scales operations, but it does not solve semantic normalization for app teams.

Director-ready answer

Crossplane is the better choice because our hardest problem is not hosting APIs for isolated workspaces. Our hardest problem is compiling one bank-owned delivery and security intent into Akamai, Cloudflare, and future provider implementations in a controlled, repeatable way.

What happens to existing Akamai Terraform?

It moves behind the control plane as an execution adapter. We keep the investment, reduce rewrite risk, and gradually replace only what becomes a bottleneck.

How agentic AI helps without becoming the trust boundary

What to tell leadership

We should use agentic AI to generate adapter code, tests, and migration mappings inside a tightly constrained framework where the platform team owns the canonical model and validation gates.

Why not write the whole thing in Python?

You can. But then you are choosing to build both the adapters and the full control-plane framework yourself.

Pure Python sounds simpler at first, but it means Chase also owns reconciliation loops, drift handling, retries, status, composition, dependency ordering, API lifecycle, and day-2 operations.

What Python is still great for

  • Provider adapters
  • Translation logic
  • Migration tools
  • Validation helpers
  • AI-generated scaffolding and tests

Why Crossplane is still the better core

Crossplane gives us the control-plane engine so our engineers can focus on the bank-specific logic that matters: the canonical model, provider mappings, policy rules, and safe rollout.
